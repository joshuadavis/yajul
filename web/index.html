YAJUL

Yet Another Java Utility Library

Browse CVS

Ever get annoyed that some basic features aren't part of the JDK?  Tired of writing the same Servlet or JDBC code over and over again?  YAJUL is made to address these concerns.

Reasons why corporate and commercial software development teams don't use Open Source libraries:

  • Open Source libraries don't have support.
    This is usually a result of the NIH, CYA, or Boat Anchor anti-patterns.  It can also be an assumption made on the bases of the cost of the libraries ($0).

    YAJUL comes with support forums and e-mail lists (like most other Open Source projects ;->).

  • Open Source libraries aren't tested.
    Sadly, this is true of many libraries.  They are tested by developers during the 'beta' phase.

    YAJUL should be different.  The YAJUL project will strive to reach 100% coverage with the unit tests that are available with the library.

  • Open Source source code is (choose expletive that means very bad or sloppy)
    Also true (ever looked at the Apache Web Server source code?).  Even some of the best projects are riddled with style problems, lack of documentation.  These are all the result of anti patterns.

    The YAJUL development process will strive to create code of the highest quality and re-useability.

Main Libraries

YAJUL is comprised of several libraries, allowing the developer to choose a minimal, lightweight set.

J2SE Library

  • Multi-threading.
    • Semaphores and POSIX-Like condvars.
    • Thread safe queue and topics.
    • ThreadPool
  • Networking.
    • Generic socket server
    • FTP server
  • String Parsing
  • I/O Utilities
    • Copying streams
    • Inverting stream direction.

GUI Library

  • MVC Components
  • Drawing utilities

Servlet Library

  • File Upload
  • XSLT Servlet
  • XML-RPC Server

J2EE

  • Simple JNDI Tree (for testing)
  • JDBC Connection Pool / DataSource objects.
